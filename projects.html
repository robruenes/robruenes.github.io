Web Programming

The Marauder's Map
If you're familiar with the world of Harry Potter, you should get a kick out of this. The Marauder's Map connects to a datastore and does an HTTP POST with your geolocation. In return, it receives the location of the last 100 students who POSTed to the datastore, along with the locations of some characters like Carmen Sandiego and a few Tufts professors. You can click on any icon displayed on the map to see specific info about that person. In addition, the map draws polylines from you to the special entries in the map, and tells you your distance in miles to them.


Visualization

Force-Directed Node Link Diagram
An environment simulating a self-structuring graph. Individual nodes can be dragged around, exerting forces on the rest of the system to move along with it

Squarified Treemaps
An implementation of a visualization first described in this paper. The visualization aims to extend the commonly used treemap method for displaying hierarchical information (like directory structures, for example). The standard treemap visualization typically results in areas of extremely thin rectangles. This extension aims to rectify the problem by generating the lay-out such that all the rectangles are approximate squares.

Theme River
An implementation of a visualization first described in this paper. As described by the abstract, the visualization is a "system that visualizes thematic variations over time across a collection of documents. The 'river' flows through time, changing width to depict changes in the thematic strength of documents temporally collocated. Themes or topics are represented as colored 'currents' flowing within the river that narrow or widen to indicate decreases or increases in the strength of a topic in associated documents at a specific point in time."

Line and Bar Chart
The line and bar chart project involved dynamic data entry and resizing of the visualization based on the input data. Data is fed into the visualization via a .csv file, and the result is a system that can toggle back and forth between a line graph and bar chart displaying the data.


Internet-Scale Distributed Systems

File Copy
In a joint effort, Luke Geneslaw and I implemented our own file copy protocol, LRFCP. We implemented both a client and server that utilize UDP to transfer the files contained within a directory on the client side to a target directory on the server side. In addition, the client and server take additional parameters known as "network nastiness" and "file nastiness," which can simulate dropped/delayed packets and read/write errors. The increased nastiness levels can be used to demonstrate our robust implementations of the client and server, in addition to confirming the validity of our end-to-end check, designed to follow the End-to-End Principle first outlined by Saltzer, Reed, and Clark.

Remote Procedure Call
In collaboration with fellow student Diego Carranza, we implemented a command line tool called RPCGenerate, which, when provided a legal IDL file, would automatically generate the proxy and stub files necessary for remoting functions through Remote Procedure Call.


Machine Structure and Assembly Language Programming

Image Compression Suite
Written in C. The image compressor utilizes discrete cosine transform and is capable of taking a PPM image and writes a file of 32-bit binary words. The decompressor is able to read one of these binary files and reproduce the original image. Our image compression/decompression suite outperformed the reference implementation across the board.

PPMtrans Image Rotator
Written in C. The image rotator, known as PPMtrans, can take in a PPM image and allow the user to rotate, flip, or transpose the provided image. The abstract purpose of this project was to teach students about locality, so the PPMtrans program also allows the user to specify whether pixels are traversed in row-major, col-major, or block-major order when the rotation process is being carried out.

Universal Machine and Related Projects
Written in C. The universal machine was easily my favorite project from this course. The machine consists of 32-bit registers and an address space utilizing 32-bit addresses, and utilizes 14 word-oriented instructions. The original version of the Universal Machine ran 50 million instructions in 4.7 seconds. In a related project on speed profiling, the UM was modified such that processes took just 20% of their original computation time to complete execution. In addition, a Macro Assembler was written for the UM, adding several instructions for those wishing to program in the UM Assembly Language, and with this tool a Reverse Polish Notation calculator was written for the Universal Machine.


Programming Languages

ImpCore
The first assignments for 105 dealt with a language called ImpCore (Imperative Core), meant to demonstrate the basic features of an imperative language. I implemented several basic functions in the language, and in addition, added support for local variables in the C interpreter for the language.

microScheme
The assignment in microScheme (a language developed to capture the most important features of Scheme) included writing the following functions in the language: map, fold, filter, take, drop, zip, and unzip.

ML
The first ML assignment involved implementing functions like foldl, foldr, zip, unzip, and flatten. In addtion, I implemented environments (mappings of names to values) two different ways; the first using lists, and the second using functions. Sequences with constant-time append, and finger-lists (with constant-time insertion deletion at the finger) were also implemented.

Type Checking
The most substantial project for this class involved implementing the Type Checkers (in ML) for both typed ImpCore and typed microScheme (versions of ImpCore and microScheme with explicit types), demonstrating knowledge of the connection between type rules, representation of types, and the ML code that made up the interpreters.
