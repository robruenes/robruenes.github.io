Internet-Scale Distributed Systems

In the Fall 2013 semester, I took COMP150-IDS: Internet-Scale Distributed Systems, taught by Noah Mendelsohn. Projects for this class were written in C++.

File Copy

In a joint effort, Luke Geneslaw and I implemented what we call "LRFCP" - the "Luke Rob File Copy Protocol." We implemented both a client and server that utilize UDP to transfer the files contained within a directory on the client side to a target directory on the server side. In addition, the client and server take additional parameters known as "network nastiness" and "file nastiness," which can simulate dropped/delayed packets and read/write errors. The increased nastiness levels can be used to demonstrate our robust implementations of the client and server, in addition to confirming the validity of our end-to-end check, designed to follow the End-to-End Principle first outlined by Saltzer, Reed, and Clark.

Remote Procedure Call

In collaboration with fellow student Diego Carranza, we implemented a command line tool called RPCGenerate, which, when provided a legal IDL file, would automatically generate the proxy and stub files necessary for remoting functions through Remote Procedure Call.
